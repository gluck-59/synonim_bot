## Телеграм-бот «Синоним»
*Ищет синонимы для слов русского языка*

https://telegram.me/synonim_bot


В коде полный бардак, стиль "функциональный говнокод" . Проект был нужен для изучения контекста работы телеграм-ботов в то время когда никаких best practices еще не было.



--------------

### Как работает ``state`` в таблице ``synonim_bot`` 

- Когда первый вызов getSynonims вернул ``state = 2``, выполняется орфографическая проверка (checkSpell), нормализуется текст (если найдено исправление) и повторяется запрос синонимов для исправленного слова.
- Если checkSpell не вернул исправления, повторного запроса не будет; результат с ``state = 2`` идёт на форматирование.
- Yandex spelling часто тупо висит, поэтому можно его отключить.
 
Откуда берётся state:
- В ``getSynonims($text)``, если парсер не нашёл синонимы (вернул пусто), устанавливается ``state`` = 2 и возвращается подсказка для пользователя.
- Если запрошенное слово есть в кэше ``synonim_cache`` возвращается ``state`` = 1.
- Если в кэше его нет, производится запрос к словарю, ответ помещается к кэш и возвращается ``state`` = 0.

Поток выполнения:
- buildSynonymResponse вызывает getSynonims, при ``state`` === 2 пытается исправить орфографию и повторяет запрос.
- formatOutput: если ``out`` пуст, принудительно ``state`` = 0 и показывается сообщение об ошибке. На практике при ``state`` = 2 ``arr`` всегда содержит подсказку, поэтому ``out`` не пуст и ``state`` остаётся = 2.
- В processMessage статистика suggest прикладывается только при ``state`` = 1; при ``state`` 2/0 ``suggest`` не отправляется.

Пограничные случаи:
- Если ``checkSpell()`` вернул исправление, но повторный ``getSynonims()`` снова пуст, ``state`` снова будет = 2. Ветка ``state`` = 0 достигается только если массив синонимов пуст вообще; текущая реализация getSynonims при неуспехе всегда возвращает подсказку в arr, поэтому ``state`` = 0 практически не встречается.
- Т.е. ``state`` это watchdog неуспешного первого поиска: при срабатывании запускается проверка орфографии и повторная попытка.
